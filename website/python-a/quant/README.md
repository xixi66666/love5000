# A 股专业多因子选股系统设计说明

> 目标：构建一套专业、完整、可运行、可回测、可迭代的多因子选股系统。
> 边界：本系统只做量化研究、风险识别、组合候选排序和复盘辅助，不构成投资建议，不承诺收益，不输出确定性买卖结论。

## 1. 系统目标

本系统面向 A 股中长期量化研究，优先服务以下目标：

1. 数据质量可靠：所有行情、财务、估值、行业、交易状态和因子数据必须记录来源、更新时间、交易日、字段含义和可用性。
2. 模型逻辑可解释：每个因子必须说明经济含义、计算公式、适用范围、滞后规则、缺失值处理和潜在失效场景。
3. 回测过程严谨：必须避免前视偏差、幸存者偏差、过拟合、错误复权、交易成本缺失、涨跌停和停牌处理缺失。
4. 结果可复现：相同数据版本、模型版本、参数配置和回测区间必须得到一致结果。
5. 系统可迭代：因子、权重、回测规则、风控约束和展示逻辑必须模块化，允许独立验证和替换。
6. 长期可运维：支持定时更新、缓存、日志、异常告警、版本记录、健康检查和结果归档。

## 3. 总体架构

```text
数据层 -> 因子层 -> 打分层 -> 回测层 -> 风控层 -> 展示层
```

每一层都必须输出可检查的中间结果，禁止只输出最终股票名单。

## 4. 数据层

### 4.1 数据层目标

数据层负责采集、清洗、对齐、校验、缓存和版本化。它决定模型上限。错误数据会导致错误因子、错误回测和错误结论。

### 4.2 核心数据类型

#### 4.2.1 股票基础信息

必须包含：

```text
股票代码
交易所
股票名称
上市日期
退市日期
证券状态
是否 ST / *ST
所属行业
总股本
流通股本
```

关键规则：

1. 历史回测时只能使用当时已经上市且可交易的股票。
2. 不能只使用当前仍上市股票，否则会产生幸存者偏差。
3. ST、退市整理、长期停牌股票必须有明确过滤或惩罚规则。

#### 4.2.2 交易日历

必须包含：

```text
交易日期
是否交易日
前一交易日
后一交易日
周度调仓日
月度调仓日
季报披露窗口
```

关键规则：

1. 所有时间序列必须以交易日历对齐。
2. 不允许用自然日简单偏移替代交易日偏移。
3. 调仓日、收益计算日、财报可用日必须明确区分。

#### 4.2.3 行情数据

日频行情至少包含：

```text
trade_date
code
open
high
low
close
pre_close
volume
amount
turnover_rate
adj_factor
limit_up
limit_down
suspend_flag
```

分钟级或 Tick 级数据第一阶段不强制，但预留扩展。

关键规则：

1. 回测收益必须明确使用前复权、后复权或不复权，不能混用。
2. 成交额和换手率用于流动性过滤，不能只看价格涨跌。
3. 涨停、跌停、停牌必须在交易执行环节处理。

#### 4.2.4 财务数据

至少包含：

```text
报告期
公告日期
更新日期
营业收入
归母净利润
扣非净利润
经营现金流
总资产
净资产
总负债
应收账款
存货
商誉
ROE
ROA
毛利率
净利率
资产负债率
```

关键规则：

1. 因子使用财务数据时必须按公告日期生效，不能按报告期直接生效。
2. 若年报在 2025-04-20 公告，则 2025-04-19 及以前不能使用该年报。
3. 财报修正、补发、更新必须保留版本或至少保留更新时间。

#### 4.2.5 估值数据

至少包含：

```text
PE_TTM
PB
PS_TTM
PCF_TTM
股息率
总市值
流通市值
```

关键规则：

1. 极端估值必须处理，例如负 PE、PE 极大值、净资产为负。
2. 估值因子要做行业内比较，避免银行、科技、医药等行业天然估值差异造成误判。

#### 4.2.6 行业与主题数据

至少包含：

```text
一级行业
二级行业
概念主题
指数成分
行业指数收益
行业成交额
```

关键规则：

1. 行业分类必须记录版本。
2. 回测时应使用当时有效的行业分类，无法获得历史行业分类时必须记录限制。
3. 行业中性化必须使用统一行业口径。

#### 4.2.7 交易约束数据

必须包含：

```text
停牌状态
涨跌停价格
是否一字涨停
是否一字跌停
是否可买
是否可卖
最小交易单位
手续费规则
印花税规则
滑点规则
```

### 4.3 数据质量要求

每批数据入库前必须做校验：

```text
字段完整性
主键唯一性
交易日连续性
价格高低关系
成交额非负
复权因子非空
公告日期不晚于入库日期
异常值分布
缺失率统计
重复记录检测
跨数据源抽样对账
```

异常数据处理策略：

1. 可修复：记录修复规则和修复前后值。
2. 不可修复：标记为不可用，不参与因子计算。
3. 数据源冲突：优先保留多个来源并做对账，不直接覆盖。
4. 缺失严重：该股票当期因子置空，并在打分层降低可用性评分。

### 4.4 数据版本

每次生成因子或回测结果时，必须记录：

```text
数据源名称
数据抓取时间
数据覆盖区间
数据版本号
代码版本号
模型配置版本
```

## 5. 因子层

### 5.1 因子层目标

因子层把原始数据转换为可比较、可解释、可验证的特征。每个因子都必须能单独回测和评估。

### 5.2 因子分类

#### 5.2.1 估值因子

目的：寻找相对便宜的股票。

候选因子：

```text
PE_TTM 倒数
PB 倒数
PS_TTM 倒数
PCF_TTM 倒数
股息率
自由现金流收益率
行业内估值分位
自身历史估值分位
```

注意事项：

1. 负 PE 不应简单视为便宜。
2. 低估值可能是价值陷阱，需要质量和盈利趋势共同过滤。
3. 周期行业估值要结合景气周期。

#### 5.2.2 成长因子

目的：识别收入、利润和现金流持续改善的公司。

候选因子：

```text
营业收入同比增长
净利润同比增长
扣非净利润同比增长
经营现金流同比增长
EPS 增长
毛利率改善
净利率改善
ROE 改善
```

注意事项：

1. 单季度高增长可能来自低基数。
2. 利润增长必须结合现金流质量。
3. 成长因子对财报公告日期非常敏感，必须防前视。

#### 5.2.3 质量因子

目的：识别盈利稳定、财务健康、造假风险较低的公司。

候选因子：

```text
ROE
ROIC
毛利率
净利率
经营现金流 / 净利润
资产负债率反向
应收账款 / 营收反向
存货 / 营收反向
商誉 / 净资产反向
盈利稳定性
```

注意事项：

1. 高 ROE 可能由高杠杆驱动。
2. 高利润但现金流差要惩罚。
3. 财务异常因子应单独输出风险解释。

#### 5.2.4 动量因子

目的：捕捉市场趋势和相对强弱。

候选因子：

```text
近 20 日收益
近 60 日收益
近 120 日收益
近 250 日收益
剔除最近 5 日后的中期动量
相对行业收益
创新高距离
均线多头排列
```

注意事项：

1. 短期反转和中期动量要分开。
2. 动量因子容易提高换手率，必须考虑交易成本。
3. 极端涨幅股票要结合波动和流动性过滤。

#### 5.2.5 波动与风险因子

目的：降低组合波动和尾部风险。

候选因子：

```text
近 20 日波动率反向
近 60 日波动率反向
Beta 反向
最大回撤反向
下行波动率反向
跳空频率反向
跌停次数反向
```

注意事项：

1. 低波动不等于低风险，可能存在流动性风险。
2. 风险因子更适合作为约束或惩罚项，不一定作为收益因子。

#### 5.2.6 流动性因子

目的：保证策略可交易，降低冲击成本。

候选因子：

```text
日均成交额
日均换手率
买卖价差
成交额稳定性
零成交天数反向
冲击成本估计反向
```

注意事项：

1. 流动性过低股票必须过滤。
2. 流动性过热可能代表拥挤交易，需要风险惩罚。

#### 5.2.7 资金流因子

目的：识别资金关注度和交易结构变化。

候选因子：

```text
主力资金净流入
北向资金持股变化
融资余额变化
成交额放大倍数
量价配合得分
龙虎榜机构席位
```

注意事项：

1. 免费资金流数据口径差异大，必须记录来源。
2. 资金流因子噪声高，不能单独作为强信号。

#### 5.2.8 事件与预期差因子

目的：捕捉公告、业绩、回购、分红、指数调整等事件影响。

候选因子：

```text
业绩预告超预期
利润预告上修
回购公告
分红提高
股权激励
定增完成
指数纳入
解禁压力反向
处罚和诉讼反向
```

注意事项：

1. 事件发生时间必须以公告发布时间为准。
2. 事件因子需要明确有效窗口，例如 5、20、60 个交易日。

### 5.3 因子处理流程

标准流程：

```text
原始因子
-> 缺失值处理
-> 极值处理
-> 方向统一
-> 标准化
-> 行业中性化
-> 市值中性化
-> 因子有效性检验
-> 保存因子版本
```

方向统一：

```text
分数越高代表越好
```

极值处理建议：

```text
MAD 去极值
分位数截尾
按行业内截尾
```

标准化建议：

```text
z-score
rank percentile
行业内 rank
```

中性化建议：

```text
行业哑变量回归残差
市值对数回归残差
行业内排序
```

### 5.4 因子有效性评估

每个因子至少输出：

```text
IC
Rank IC
IC 均值
IC 标准差
ICIR
分组收益
多空收益
最大回撤
换手率
行业暴露
不同市场环境表现
样本外表现
```

因子上线条件：

1. 有明确经济含义。
2. 样本内和样本外方向基本一致。
3. 分组收益具有单调性或合理解释。
4. 不依赖少数极端年份。
5. 加入组合后改善收益、回撤或稳定性。

## 6. 打分层

### 6.1 打分层目标

打分层负责把多个因子合成为股票综合评分，并生成可解释排序。

### 6.2 基础评分方式

第一阶段建议使用可解释加权模型：

```text
综合得分 =
估值得分 * w_value
+ 成长得分 * w_growth
+ 质量得分 * w_quality
+ 动量得分 * w_momentum
+ 风险得分 * w_risk
+ 流动性得分 * w_liquidity
```

第一版六维默认权重作为 `v0.1 baseline`：

```text
估值：15%
成长：15%
质量：25%
动量：20%
风险：15%
流动性：10%
```

默认权重只作为 `v0.1 baseline`，不是固定真理，必须通过回测、样本外验证和年度分段表现评估后再调整。

资金流和事件因子第一版不进入核心六维权重，后续作为增强因子、风险标签或实验维度引入。引入前必须先解决数据口径、可用日、噪声和样本外稳定性问题。

### 6.3 股票池过滤

打分前必须先过滤：

```text
未上市或已退市
上市未满 120 个交易日
ST / *ST
停牌
退市整理
近 20 日日均成交额过低
当日无法买入的一字涨停
财务数据严重缺失
重大异常风险标记
```

过滤规则必须可配置，不能写死在因子计算中。

### 6.4 缺失值处理

缺失值不能随意填 0。

建议规则：

1. 因子缺失率低：行业中位数填充，并记录填充标记。
2. 因子缺失率高：该因子当期不参与该股票评分。
3. 股票缺失因子过多：降低可用性评分或剔除。
4. 关键财务字段缺失：禁止使用相关财务因子。

### 6.5 输出内容

每次打分必须输出：

```text
trade_date
code
name
industry
total_score
rank
factor_scores
factor_raw_values
missing_fields
risk_flags
data_version
model_version
```

展示时不能只展示总分，必须展示主要贡献因子和风险扣分原因。

## 7. 回测层

### 7.1 回测层目标

回测层用于验证策略在历史环境中的表现，目标不是制造漂亮曲线，而是尽量接近真实交易。

### 7.2 回测基本参数

每次回测必须显式配置：

```text
回测区间
初始资金
股票池
调仓频率
持仓数量
单票权重上限
行业权重上限
买入价格规则
卖出价格规则
手续费
印花税
滑点
涨跌停处理
停牌处理
复权方式
基准指数
```

### 7.3 时间线规则

必须遵守：

```text
T 日收盘后计算信号
T+1 日按规则买入或卖出
T+1 日以后计算持仓收益
```

禁止：

```text
使用 T 日收盘价生成信号后，又假设在 T 日收盘价成交
使用未来财报数据生成过去信号
使用未来指数成分生成过去股票池
```

### 7.4 财务数据生效规则

财务数据必须按公告日期生效：

```text
可用日期 = 公告日期后的下一个交易日
```

若数据源没有公告日期，该字段不得用于严谨回测，除非文档明确标记为低可信实验。

### 7.5 股票池规则

回测股票池必须是历史动态股票池：

```text
每个交易日只包含当时已上市、未退市、可交易的股票。
指数成分股必须使用历史成分。
行业分类必须尽量使用历史口径。
```

若无法获得历史成分，则必须在回测报告中标注：

```text
存在幸存者偏差风险。
```

### 7.6 成交价格规则

第一阶段建议：

```text
信号日：T 日收盘后
买入日：T+1 日
成交价：T+1 日 VWAP 或开盘价加滑点
卖出日：T+1 日
成交价：T+1 日 VWAP 或开盘价减滑点
```

如果没有分钟数据，使用开盘价或均价近似，但必须记录假设。

成交执行必须补充以下细则：

```text
价格优先级：有 VWAP 使用 VWAP；没有 VWAP 使用开盘价；开盘价缺失时该笔订单不成交。
买入成交价 = 基准成交价 * (1 + 滑点率)
卖出成交价 = 基准成交价 * (1 - 滑点率)
成交价格不得突破当日涨跌停价格边界。
买入数量必须按 100 股整数手向下取整。
卖出数量必须按当前可用持仓数量向下取整，不允许卖出超过持仓。
现金不足时按目标金额、整数手和费用重新计算最大可买数量。
单笔买入金额不足 1 手时跳过该订单并记录原因。
```

回测引擎必须输出订单级执行结果：

```text
order_id
trade_date
signal_date
code
side
target_amount
target_quantity
filled_quantity
filled_price
commission
tax
transfer_fee
slippage_cost
impact_cost
order_status
reject_reason
```

### 7.7 交易成本规则

A 股基础成本至少包含：

```text
买入佣金
卖出佣金
卖出印花税
过户费
滑点
冲击成本
```

建议默认：

```text
佣金：万 2.5，单笔最低 5 元可配置
印花税：按当前规则配置，不写死历史全局常量
滑点：双边 5 到 20 个基点可配置
冲击成本：与成交额占比相关
```

手续费和税费规则可能变化，必须版本化。

每次回测必须记录成本模型版本：

```text
cost_model_version
commission_rate
min_commission
stamp_tax_rate
transfer_fee_rate
slippage_bps
impact_cost_rule
effective_start_date
effective_end_date
```

成本规则不能写死在回测计算里，必须由配置传入。历史规则变化时，应新增成本模型版本，不直接覆盖旧版本。

### 7.8 涨跌停规则

必须处理：

```text
涨停不可买入
跌停不可卖出
一字涨停不可买入
一字跌停不可卖出
停牌不可交易
```

若调仓日无法卖出：

```text
继续持有
记录未成交订单
下一个交易日继续尝试或按配置取消
```

若调仓日无法买入：

```text
保留现金
或按候补名单顺延买入
```

规则必须配置化。

未成交订单处理规则：

```text
默认重试次数：1 个交易日
卖出失败：继续持有，下一交易日继续尝试一次；再次失败后按配置继续挂单或取消。
买入失败：默认保留现金；如果启用候补名单，则按排名顺延尝试买入。
部分成交：记录已成交数量，剩余数量进入未成交订单或按配置取消。
停牌订单：不成交，记录 suspend_reject。
涨停买入订单：不成交，记录 limit_up_reject。
跌停卖出订单：不成交，记录 limit_down_reject。
现金不足订单：重新按可用现金计算整数手；仍不足 1 手则取消。
```

每个未成交订单必须记录：

```text
order_id
first_attempt_date
last_attempt_date
retry_count
remaining_quantity
blocked_reason
next_action
```

```text

单票上限 5%

```

### 7.11 数据可用日规则

回测中所有数据必须有明确可用日，禁止默认“抓到即历史可用”。

```text
行情数据可用日：交易日收盘后，最早用于下一交易日信号或成交。
财务数据可用日：公告日期后的下一个交易日。
估值数据可用日：数据源标注日期收盘后，最早用于下一交易日信号。
行业分类可用日：分类生效日；缺少历史生效日时必须标注口径限制。
指数成分可用日：指数公司公告或生效日；缺少历史成分时必须标注偏差。
事件数据可用日：公告发布时间后的下一个交易日。
```

若某个字段没有可靠公告日、生效日或发布时间，处理方式只能二选一：

```text
不进入严谨回测，只允许作为低可信实验字段。
进入回测但必须在报告中显著标注数据可用日不完整带来的前视偏差风险。
```

### 7.12 样本划分

禁止只看一段历史。

建议：

```text

测试期：用于最终评估

```

示例：

```text

```

### 7.13 过拟合控制

```text

```

### 7.14 偏差披露规则

回测报告必须主动披露以下偏差和限制，不允许只输出收益指标：

```text
幸存者偏差：是否包含历史退市、暂停上市、长期停牌股票。
前视偏差：财务、估值、事件、行业、指数成分是否有可靠可用日。
复权偏差：使用前复权、后复权或不复权，以及收益计算口径。
成交偏差：是否使用分钟 VWAP；若使用开盘价或均价近似，需要说明。
成本偏差：手续费、税费、滑点、冲击成本是否按历史版本化规则计算。
流动性偏差：是否限制成交额占比、是否处理低流动性股票。
涨跌停偏差：是否处理涨停不可买、跌停不可卖、一字板和停牌。
行业口径偏差：是否使用历史行业分类；若使用当前行业，需要说明。
指数成分偏差：是否使用历史成分；若使用当前成分，需要说明。
数据源偏差：免费数据源延迟、字段变化、历史修正和缺失情况。
```

偏差披露必须包含：

```text
bias_type
severity
affected_period
affected_fields
impact_description
mitigation
```

## 8. 风控层

### 8.1 风控层目标

风控层不负责预测收益，而是限制模型犯大错。

### 8.2 组合约束

建议默认约束：

```text
单票权重 <= 5%
单行业权重 <= 30%
ST 股票权重 = 0
停牌股票不可新买
近 20 日日均成交额低于阈值不可新买
单日换手率上限
组合现金下限
最大回撤预警线
```

### 8.3 风格暴露控制

需要监控：

```text
市值暴露

动量暴露
波动率暴露
行业暴露
Beta 暴露
流动性暴露
```

如果组合收益主要来自某个隐含暴露，必须在报告中说明。

### 8.4 个股风险标签

至少输出：

```text
ST / 退市风险

商誉高风险
现金流异常
应收账款异常
存货异常
财报延期

诉讼风险
解禁压力
流动性不足
连续涨停后追高风险
高位放量滞涨风险
```

### 8.5 风控动作

风控动作分为：

```text
过滤：禁止进入股票池
降权：降低综合分
限权：限制最大持仓
预警：允许持有但必须提示
复核：需要人工确认
```

系统不应因为单一风险标签直接输出确定性结论，而应说明证据和处理方式。

## 12. 已知限制

第一阶段必须明确以下限制：

1. 免费数据源可能存在延迟、字段口径变化、历史修正和缺失。
2. 没有完整历史指数成分和历史行业分类时，回测可能存在偏差。
3. 没有分钟数据时，成交价和滑点只能近似。
4. 单纯历史回测无法证明未来收益。
5. A 股制度、交易规则、市场风格和监管环境会变化。

## 13. 参考原则

本设计遵循以下行业常见原则：

1. 回测必须防止前视偏差、幸存者偏差、过拟合，并纳入交易成本和市场约束。
2. 因子模型不仅要看收益，还要看风险暴露、行业暴露、换手率和稳定性。
3. 组合构建必须把收益预测、风险控制和交易执行分开处理。
4. 所有投资研究输出都必须包含证据、置信度、反证条件和风险提示。
