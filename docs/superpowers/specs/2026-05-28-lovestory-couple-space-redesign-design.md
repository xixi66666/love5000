# lovestory 情侣空间重做设计

## 背景

`lovestory` 当前首页是一个单页静态页面，包含登录校验、小游戏入口、回忆卡片、照片墙、照片上传、Lightbox、秘密留言板和里程碑等能力。现有页面的问题主要集中在三类：

- 视觉层级较散，多个模块像是顺序堆叠，整体温馨感和空间感不足。
- 回忆卡片已有 `flipCard` 函数和翻转样式，但卡片自身的 `transform`、悬浮状态和翻转状态叠加，容易导致反转效果不稳定。
- 部分文案存在编码异常痕迹，影响页面质感和可维护性。

本次改造选择“大幅重做成情侣空间”，保留现有功能，重新组织页面结构和交互体验。

## 目标

- 把首页改成更完整的情侣空间体验，而不是模块堆叠页。
- 页面风格以温柔、日记、相册、暖光为主，并加入少量浪漫仪式感。
- 修复并升级反转卡片功能，使其在桌面端和移动端都可稳定使用。
- 保留现有照片墙筛选、Lightbox、上传照片、删除照片、秘密留言板、小游戏入口等功能。
- 提升移动端可读性和可点击性，避免文字、按钮、卡片相互挤压或遮挡。

## 非目标

- 不改动后端接口路径和响应格式。
- 不新增数据库表或调整 MyBatis Mapper。
- 不引入 Vue、React 或新的前端构建流程。
- 不把页面拆成多文件前端工程；本轮仍优先维护当前静态 HTML 交付方式。
- 不真实调用外部 OSS 或生产数据库作为验证依赖。

## 页面结构

页面仍使用 `lovestory/src/main/resources/static/index.html` 作为主入口，整体结构调整为：

1. 沉浸式首页
   - 展示 `xixi & 5000`、恋爱计时、开始日期和主要入口。
   - 使用现有真实照片作为首屏视觉信号，叠加柔和暖色遮罩。
   - 首屏底部露出下一块内容，避免像单独落地页。

2. 固定导航
   - 桌面端显示分区跳转：小游戏、回忆卡片、照片墙、里程碑、秘密留言、上传照片。
   - 移动端使用可横向滚动的紧凑导航，避免遮挡主体内容。
   - 导航只负责跳转和打开已有功能，不承载复杂业务逻辑。

3. 约会小角落
   - 保留 `snake.html`、`tic-tac-toe.html`、`match3.html` 三个小游戏入口。
   - 视觉上从“突兀游戏卡片”改成轻量的情侣空间模块。

4. 互动回忆卡片
   - 每张卡片包含正面和背面。
   - 正面：照片、日期、标题、简短提示。
   - 背面：回忆标题和正文。
   - 支持点击、触摸、键盘 Enter 和 Space 翻转。
   - 洗牌功能保留，但需要使用稳定的 DOM 顺序和状态重置方式。

5. 照片墙
   - 保留分类筛选、动态加载上传照片、Lightbox、删除上传照片等现有行为。
   - 视觉改成相册墙：照片比例稳定、说明文字清晰、筛选按钮统一。
   - 上传失败、加载失败、空数据时有明确可读的提示。

6. 恋爱里程碑
   - 保留当前里程碑内容，改成更精致的时间线或纪念节点。
   - 节点文案保持简洁，避免在卡片内塞过多大字号文字。

7. 秘密留言与上传入口
   - 保留秘密留言板功能，但入口更清晰，不只依赖双击隐藏操作。
   - 上传照片保留悬浮入口和弹窗，但按钮、预览区和进度条视觉统一到新风格。

## 视觉系统

推荐风格为“温柔日记感 + 少量浪漫仪式感”：

- 主背景：奶白、浅粉、暖米色，不做单一大面积纯粉。
- 强调色：柔和玫瑰色和暖金色，用于按钮、计时数字、卡片重点信息。
- 文本色：深棕灰或深玫瑰灰，保证正文可读性。
- 卡片：半透明或白色纸张感，边角不超过 16px，阴影轻柔。
- 图片：真实照片优先，使用稳定比例和柔和边框，不用纯装饰 SVG 替代照片。
- 动效：只用于翻卡、按钮反馈、模块进入和弹窗，不做过多漂浮装饰。
- 字体：继续使用中文无衬线字体，标题可以保留少量手写感字体，但正文以清晰可读为先。

## 反转卡片交互设计

现有 `.memory-card` 同时承担布局定位、悬浮变换和翻转变换，容易互相覆盖。新结构改为：

- `.memory-card`：负责卡片外层位置、点击、焦点和层级。
- `.memory-card-inner`：负责 `rotateY(180deg)` 翻转。
- `.card-front` 和 `.card-back`：负责正反两面内容，使用 `backface-visibility: hidden`。

交互规则：

- 点击卡片时切换 `is-flipped`。
- Enter 或 Space 聚焦卡片时也切换 `is-flipped`。
- 洗牌时先移除所有翻转状态，再重新排列卡片。
- 移动端不依赖 hover 展示关键信息。
- 对 `prefers-reduced-motion: reduce` 用户，取消 3D 动画或改为快速淡入淡出。

## 数据与接口

本次是前端静态页面改造，不调整接口：

- `GET /api/photos`
- `POST /api/photos/upload`
- `DELETE /api/photos/{id}`
- `GET /api/messages`
- `POST /api/messages`
- `DELETE /api/messages`

前端继续按现有接口调用。新增或重构 JS 时，需要保持这些接口的请求方式、参数和错误处理不变。

## 错误处理

- 照片加载失败：照片墙显示可读提示，不让页面空白。
- 上传失败：弹窗内恢复按钮状态，并给出明确失败原因。
- 留言加载失败：留言板显示“留言加载失败，请稍后重试”。
- 删除失败：保留当前照片，提示失败原因。
- 外部图片无法加载：卡片和照片墙保留占位背景，避免布局塌陷。

## 可访问性与响应式

- 所有可点击卡片和按钮都需要清晰焦点态。
- 卡片支持键盘操作。
- 移动端触控目标不小于 44px。
- 页面不产生横向滚动。
- 首屏、卡片区、照片墙在 375px 宽度下文字不溢出。
- 图像必须有有意义的 `alt` 文案。
- 弹窗打开时尽量保持视觉焦点明确，Esc 可以关闭已有弹窗。

## 实现边界

本次实现优先集中在：

- `lovestory/src/main/resources/static/index.html`

如实现过程中发现单文件过大严重影响维护，可以在不改变运行方式的前提下拆出：

- `lovestory/src/main/resources/static/css/lovestory.css`
- `lovestory/src/main/resources/static/js/lovestory.js`

拆分只作为可维护性改进，不改变页面入口和接口行为。

## 验证方式

实现完成后至少验证：

- 页面能通过 `lovestory` Spring Boot 服务正常访问。
- 登录校验仍按现有逻辑工作。
- 恋爱计时正常更新。
- 小游戏入口能打开对应页面。
- 回忆卡片点击、键盘翻转、洗牌都正常。
- 照片筛选正常。
- Lightbox 打开、切换、关闭正常。
- 上传弹窗打开、选择文件、校验和按钮状态正常。
- 秘密留言板打开、关闭、输入计数、发送和清空行为正常。
- 375px 移动端和桌面端布局无明显重叠、溢出和不可点击区域。

推荐命令：

```powershell
mvn -pl lovestory -am spring-boot:run -Dspring-boot.run.main-class=com.ycxandwuqian.love.LovestoryApplication
```

访问：

```text
http://127.0.0.1:8081/
```

## 风险

- 当前页面包含大量内嵌 CSS 和 JS，改造时需要避免误删上传、留言、Lightbox 等已有功能。
- 页面中已有部分编码异常文案，修复时要注意不要破坏 HTML 结构。
- 外部 OSS 图片依赖网络可用性，验证视觉时需要准备图片加载失败的兜底状态。
- 如果只修视觉但不整理卡片 transform 结构，反转卡片问题可能再次出现。
